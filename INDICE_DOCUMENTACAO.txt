🎨 PADRÕES PRONTOS
───────────────────────────────────────────────────────────────────────────────

📄 PADROES_CONFIGURACAO.md
   └─ 5 Padrões prontos para usar
   
   Padrão 1: Adicionar campo simples
   ├─ Passo a passo
   ├─ Exemplo: signatureResponsiblePhone
   └─ Checklist
   
   Padrão 2: Adicionar campo secret (encriptado)
   ├─ Passo a passo com encryptSecret()
   ├─ Exemplo: API key
   └─ Checklist
   
   Padrão 3: Adicionar toggle de feature
   ├─ Passo a passo
   ├─ Exemplo: featurePsychologistScheduling
   └─ Checklist
   
   Padrão 4: Adicionar configuração de integração
   ├─ Múltiplas variáveis relacionadas
   ├─ Exemplo: webhook da PF (URL + secret + toggle)
   └─ Checklist
   
   Padrão 5: Adicionar enum/select
   ├─ Valores pré-definidos
   ├─ Exemplo: paymentMethod (pix, creditcard, boleto)
   └─ Checklist
   
   Tempo: 30 min (leitura), ilimitado (referência)
   Leia quando: Quer implementar nova configuração
   Inclui também:
     • Fluxo de desenvolvimento típico
     • Referência de tipos
     • Erros comuns
     • Boas práticas
